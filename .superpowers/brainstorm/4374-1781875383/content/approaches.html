公众号编辑器 — 拖拽框架方案对比

你倾向替换为完整的页面构建器框架，以下是 3 个可行方案

方案 A：GrapesJS + Vue 3 集成

优势

  • 最成熟的开源页面构建器（23k+ stars）
  • 内置画布、模块库、样式管理器、属性面板
  • 有官方 Vue 3 wrapper（grapesjs-vue）
  • 大厂验证（多家 CMS 使用）

劣势

  • 非 Vue 原生（基于 jQuery 的 DOM 操作）
  • 与 Pinia 状态管理有冲突风险
  • 现有模块需重写为 GrapesJS 组件
  • 学习曲线较陡

方案 B：tmagic-editor（腾讯开源）

优势

  • Vue 3 原生开发，技术栈完全匹配
  • MIT 协议，腾讯团队维护
  • 支持自定义组件系统
  • 天然适配移动端/H5
  • 配置驱动，扩展性好

劣势

  • 主要用于页面搭建，偏重
  • 需学习其组件/插件体系
  • 社区相对 GrapesJS 小
  • 文档以中文为主（对你反而是优势）

方案 C：渐进式重构（推荐）

vue-draggable-plus（已有） + 抽象模块注册机制

优势

  • 零额外依赖（vue-draggable-plus 已安装）
  • 保留现有 Vue 组件，无需重写
  • 逐步迁移，风险最低
  • 解决 A（拖拽）+ B（样板代码）两项痛点

劣势

  • 仍需自己维护属性面板
  • 不如完整框架功能丰富
  • 需自行实现部分交互细节

我的推荐：方案 C（渐进式）

理由是：

  • vue-draggable-plus 已安装——零成本开始改进拖拽体验
  • 你已有 10 种模块的完整 Vue 组件，全部重写为 GrapesJS/tmagic 组件工作量很大
  • 公众号排版编辑器的需求相对固定（不是通用页面构建器），完整框架反而带来不必要的复杂度
  • 通过抽象 模块注册机制，新增模块可以减少大量样板代码

但如果你的目标不仅是拖拽改善，还希望未来扩展更多可视化编辑能力，tmagic-editor 是 Vue 3 项目的最佳全框架选择。
