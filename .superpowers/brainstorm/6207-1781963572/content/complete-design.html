完整设计总览

模板系统改造 — 数据模型 + 创建流程 + 编辑器交互

1. 数据模型 ✓

Layout.content → 从 StyleDefinitions 改为完整 Document JSON
Article.layout_id → 关联当前模板，已有此字段无变动
示例文章 → 前端预设 Document，类似 loadTestData() 扩展为多篇

2. 文章创建流程 ✓

弹窗包含：标题 + 示例文章（单选标签）+ 模板（可选）+ 创建并编辑
数据流：示例文章 Document → 按 type 匹配模板模块 → 合并 styles（不合并 props）

3. 编辑器按模块挑样式 ✓

属性面板底部新增"模板"分区 → 显示关联模板中同类型模块的样式摘要 → 点击"应用"只合并 styles

4. 工具栏改动

"🎨 切换模板" → 改为关联模板选择（仅改变 layout_id，不自动应用样式）
"📋 加载示例" → 移除该按钮

关联模板后，用户到属性面板中按模块逐个应用样式。

5. 模板保存逻辑

模板模式保存：直接存 JSON.stringify(documentStore.document) 到 layout.content
不再需要现在那种"遍历收集 per-type styles"的逻辑 — 更简单。

6. 涉及改动文件清单

前端
┣ services/api.ts — Layout 接口 styles→document
┣ stores/document.ts — applyTemplateStyles 适配新模型 + 增加 per-module 方法
┣ pages/ArticleListPage.vue — 创建弹窗增加示例文章选择
┣ pages/EditorPage.vue — 切换模板逻辑 + 移除加载示例 + 跟踪 layout_id
┗ components/PropertyPanel.vue — 新增"模板"分区

后端
┣ model/layout.go — CreateLayoutRequest.Content 去掉 binding:"required"（模板创建时无文档）
┗ （其余无改动 — content 字段本身就是 string，只是存的内容格式变了）
共约 5 个前端文件 + 1 个后端文件改动
